"정답 프롬프트를 외우는 것"이 아니라, "내 생각을 프롬프트로 옮기는 방법"을 배웁니다.
코딩을 몰라도 됩니다. 화면을 말로 설명할 수 있으면, AI가 만들어 줍니다.

🏠 비유하면: 인테리어 업체에 "거실은 밝은 톤, 소파는 L자형, TV는 벽걸이로" 설명하듯이, AI에게 원하는 화면을 말로 설명하면 됩니다!

3

실습 시작
1

뼈대 잡기

"일단 화면을 띄워보자"
⏱️ 10분

🧠 이렇게 생각해보세요

1
눈으로 그리기: 우리 팀 대시보드를 상상해봅시다 💭 "상단에 로고와 팀명, 그 아래에 메뉴 탭들, 탭을 누르면 내용이 바뀌는 화면"
2
조각으로 나누기: 가장 큰 뼈대부터 잡아봅시다 💭 "① 헤더(로고+팀명) ② 탭 메뉴(부재료실적, 등급별현황) ③ 각 탭의 빈 내용"
3
포인트: 처음부터 완벽하게 안 해도 됩니다. "빈 껍데기"를 먼저 만들고, 나중에 하나씩 채워 넣는 거예요!
📋 AI에게 보낼 프롬프트
동국제강 원료기획팀에서 사용할 업무 대시보드 웹 페이지를 만들어줘. 하나의 HTML 파일로 만들어줘. 상단에 진한 남색 배경의 헤더를 넣고, 왼쪽에 "DK 동국제강" 로고 텍스트, 오른쪽에 오늘 날짜와 "동국제강 원료기획팀 | OOO님" 텍스트를 넣어줘. 헤더 아래에 탭 메뉴 2개를 만들어줘: - 부재료실적 모니터링 - 등급별현황/수입관리 탭을 클릭하면 해당 내용이 보이고, 다른 탭 내용은 숨겨지게 해줘. 지금은 각 탭에 "여기에 OO 내용이 들어갑니다"라고만 표시해줘. 전체 배경은 밝은 회색(#F4F6F9), 헤더는 진한 남색(#1B2A4A), 포인트 색상은 오렌지(#E67E22)로 해줘.

✅ 확인 체크리스트

화면에 네이비색 헤더가 보이나요?
2개 탭을 클릭하면 내용이 바뀌나요?

💬 강사 한마디

지금 결과물은 HTML(구조) + CSS(디자인) + JavaScript(탭 전환 동작)의 조합입니다. AI가 세 가지를 한번에 만들어준 거예요! 이 세 단어를 몰라도 전혀 상관없습니다.

2

KPI 카드 만들기

"핵심 지표를 한눈에!"
⏱️ 10분

🧠 이렇게 생각해보세요

1
눈으로 그리기: 대시보드 상단에 핵심 숫자가 보이면 좋겠죠? 💭 "연간 목표, 누계 실적, 달성률, 인천/포항 비율 — 4개 카드가 나란히"
2
핵심 포인트: 처음에는 데이터가 없으니 "데이터를 입력해야 반영됩니다"라고 안내 메시지를 보여줍니다. 💭 "나중에 데이터를 넣으면 자동으로 숫자가 채워지게 하자"
📋 AI에게 보낼 프롬프트
첫 번째 탭 "부재료실적 모니터링" 상단에 KPI 카드 4개를 만들어줘. 4개 카드 구성: - 연간 목표 (만 톤 단위) - 누계 실적 (만 톤 단위) - 계획 대비 달성률 (%) - 인천/포항 실적비 (비율) 중요한 점: 1. 최초 생성 시 카드는 비어있는 상태로 표시해줘 2. 각 카드에 "데이터를 입력해야 반영됩니다"라는 안내 문구를 보여줘 3. 나중에 데이터가 입력되면 안내 문구 대신 계산된 숫자가 표시되도록 구조를 잡아줘 4. 달성률이 100% 이상이면 "초과 달성" 표시를 오렌지색으로 넣어줘 카드 디자인: - 흰색 배경에 둥근 모서리 (border-radius: 12px) - 큰 숫자는 굵고 크게 (font-size: 28px, font-weight: 800) - 설명은 작은 회색 글씨 (font-size: 13px, color: #6B7280) - 4개가 가로로 나란히, 화면 좁으면 2×2로

✅ 확인 체크리스트

KPI 카드 4개가 보이나요?
"데이터를 입력해야 반영됩니다" 안내가 표시되나요?
4개 카드가 가로로 나란히 배치되나요?

💬 강사 한마디

아직 데이터가 없지만 자리를 먼저 잡아두는 것이 핵심입니다. "빈 껍데기 → 데이터 연결"의 순서로 만들면 훨씬 안정적으로 완성할 수 있어요!

3

차트 영역 만들기

"차트 자리를 먼저 잡아두자"
⏱️ 10분

🧠 이렇게 생각해보세요

1
눈으로 그리기: KPI 카드 아래에 차트 2개가 나란히 있으면 좋겠죠? 💭 "왼쪽에 월별 계획 대비 실적 막대 차트, 오른쪽에 거래처 납품 추이 꺾은선 차트"
2
핵심 포인트: 데이터 입력 전에는 빈 차트 영역만 보여주고, 데이터가 들어오면 자동으로 차트가 그려지게! 💭 "데이터를 저장하면 차트가 자동으로 그려지는 구조를 먼저 만들자"
📋 AI에게 보낼 프롬프트
KPI 카드 아래에 차트 2개를 나란히 넣어줘. 차트는 Chart.js 라이브러리를 CDN으로 불러와서 사용해줘. 왼쪽 차트 (너비 50%): "월별 계획 대비 실적" - 막대 차트 - 가로축: 1월~12월 - 계획 합계(인천+포항): 네이비색 테두리 막대 - 실적 합계(인천+포항): 오렌지색 채운 막대 - 만톤 단위로 표시 오른쪽 차트 (너비 50%): "월별 주요 거래처 납품 추이" - 꺾은선 차트 - 거래처 납품 데이터에서 입고량 기준 상위 Top 3 거래처를 자동으로 추출해서 표시 - 거래처명과 색상은 데이터 기반으로 동적 할당 중요한 점: 1. 데이터가 저장되기 전까지는 차트 영역에 "데이터를 입력하면 차트가 표시됩니다"라는 안내를 보여줘 2. localStorage에 데이터가 있으면 자동으로 읽어서 차트를 그려줘 3. 화면이 좁아지면 차트가 위아래로 배치되게 해줘

✅ 확인 체크리스트

차트 영역 2개가 나란히 보이나요?
"데이터를 입력하면 차트가 표시됩니다" 안내가 보이나요?

💬 강사 한마디

Chart.js는 인터넷에서 무료로 가져와 쓸 수 있는 차트 도구입니다. CDN이라는 방식으로 인터넷 주소 한 줄이면 사용 가능해요. 아직 빈 상태지만, 다음 단계에서 데이터를 넣으면 바로 차트가 나타납니다!

4

월별 계획 대비 실적 데이터 입력 ⭐

"엑셀 데이터를 붙여넣자!" — 오늘의 핵심
⏱️ 15분

🧠 이렇게 생각해보세요

1
문제 인식: KPI 카드와 차트가 비어있죠? 이제 엑셀 데이터를 연결할 차례입니다! 💭 "엑셀에서 복사 → 붙여넣기 → KPI와 차트가 자동으로 채워지면 좋겠다"
2
핵심: 엑셀 구조와 동일한 그리드를 만들어서, 데이터를 통째로 붙여넣기 할 수 있게 하는 거예요. 💭 "4줄 × 12열 그리드 → 엑셀에서 Ctrl+C → 그리드에 Ctrl+V"
3
포인트: 데이터를 저장하면 KPI 카드와 차트가 자동으로 갱신됩니다. 데이터 → KPI → 차트가 하나로 연결!
📋 AI에게 보낼 프롬프트
차트 아래에 "월별 계획 대비 실적" 데이터 입력 영역을 만들어줘. 엑셀에서 데이터를 복사해서 바로 붙여넣을 수 있는 그리드 형태로 만들어줘. 그리드 구조 (엑셀과 동일하게): - 행 4개: 인천 계획 / 인천 실적 / 포항 계획 / 포항 실적 - 열 12개: 1월 ~ 12월 - 왼쪽에 행 이름(인천 계획, 인천 실적 등)을 고정 표시 핵심 기능: 1. 엑셀에서 4행 × 12열 데이터를 복사해서 그리드에 붙여넣기(Ctrl+V) 가능 2. 각 셀을 직접 클릭해서 수정도 가능 3. "적용" 버튼: 붙여넣은 값을 그리드에 반영하고 KPI 카드와 차트를 즉시 갱신 4. "저장" 버튼: 적용된 데이터를 localStorage에 저장해서 새로고침해도 유지 5. 저장 시 KPI 카드 4개가 자동 갱신: - 연간 목표: (인천계획 + 포항계획) 합계를 만 톤으로 변환 - 누계 실적: (인천실적 + 포항실적) 합계를 만 톤으로 변환 - 달성률: (실적합계 / 계획합계) × 100% - 인천/포항 비율: 인천실적합계 vs 포항실적합계 6. 저장 시 "월별 계획 대비 실적" 차트도 자동 갱신 7. 페이지를 열 때 저장된 데이터가 있으면 그리드에 자동 표시 8. "초기화" 버튼으로 그리드 데이터 전체 삭제 가능
📊 테스트용 데이터 (숫자만 드래그해서 복사하거나 복사 버튼 클릭)

✅ 확인 체크리스트

그리드에 테스트 데이터를 붙여넣을 수 있나요?
"적용" 버튼을 누르면 KPI 카드와 차트가 갱신되나요?
"저장" 버튼을 누르면 새로고침해도 데이터가 유지되나요?

💬 강사 한마디

이 단계가 핵심입니다! "엑셀 구조와 동일한 그리드"를 만들어서 붙여넣기하면, KPI 카드와 차트가 한꺼번에 갱신됩니다. 데이터 → 지표 → 시각화가 하나로 연결되는 거예요!

월별 누계 실적 상세

"데이터를 테이블로 정리하자"
⏱️ 10분

🧠 이렇게 생각해보세요

1
눈으로 그리기: Step 4에서 입력한 데이터를 보기 좋은 테이블로 정리하면 좋겠죠? 💭 "월별로 인천/포항 합계, 누계, 달성률을 한눈에 볼 수 있는 상세 테이블"
2
포인트: Step 4에서 저장한 데이터를 자동으로 읽어서 테이블을 만드는 거예요. 데이터를 다시 입력할 필요 없음!
📋 AI에게 보낼 프롬프트
데이터 입력 그리드 아래에 "월별 누계 실적 상세" 테이블을 추가해줘. 이 테이블은 앞에서 저장한 계획/실적 데이터를 자동으로 읽어서 표시해줘. 데이터가 없으면 "Step 4에서 데이터를 먼저 입력해주세요"라고 안내해줘. 테이블 구성: - 열: 월 | 인천계획 | 인천실적 | 포항계획 | 포항실적 | 합계계획 | 합계실적 | 달성률(%) - 행: 1월~12월 + 합계 행 - 달성률이 100% 이상이면 셀 배경을 연한 초록색, 100% 미만이면 연한 빨간색으로 - 합계 행은 굵은 글씨와 구분선으로 강조 - 숫자는 천 단위 콤마 표시 (예: 120,000) 데이터가 저장/갱신되면 이 테이블도 자동으로 갱신되게 해줘.

✅ 확인 체크리스트

월별 상세 테이블이 보이나요?
달성률에 따라 색상이 다르게 표시되나요?
합계 행이 강조되어 보이나요?
Step 4에서 데이터를 바꾸면 테이블도 자동 갱신되나요?

💬 강사 한마디

한 번 저장한 데이터가 KPI 카드, 차트, 테이블에 동시에 반영되는 것을 확인하세요! 이것이 "데이터 중심 대시보드"의 핵심입니다. 하나의 데이터 소스로 여러 뷰를 만드는 거예요.

6

거래처 납품 실적 데이터 (인천공장) ⭐

"2만 건 넘는 실적 데이터를 붙여넣자!"
⏱️ 15분

🧠 이렇게 생각해보세요

1
문제 인식: 인천공장의 납품 실적 데이터는 연 2만 건이 넘습니다. 이 대량 데이터를 관리할 수 있어야 해요! 💭 "엑셀과 동일한 5열 구조의 그리드를 만들고, 대량 데이터를 한번에 붙여넣기"
2
조각으로 나누기: 대량 데이터를 다루려면 특별한 기능이 필요합니다 💭 "① 엑셀과 동일한 그리드 ② 대량 붙여넣기 ③ 페이지네이션 ④ 요약 정보"
3
포인트: 2만 건을 한 화면에 다 보여주면 느려집니다. 100건씩 페이지 나누기가 필수!
📋 AI에게 보낼 프롬프트
"거래처 납품 실적관리 데이터 (인천공장)" 영역을 추가해줘. 엑셀의 구조와 동일하게 그리드를 구성해줘. 그리드 헤더 5개: - 전기일 / 공급업체명 / 검수등급명(품명) / 입고량 / 입고금액 핵심 기능: 1. "납품 데이터 불러오기" 버튼 → 클릭하면 큰 텍스트 입력창(textarea) 표시 2. 엑셀에서 5열 × 수만 행 데이터를 복사해서 한번에 붙여넣기 가능 3. "반영" 버튼으로 탭 구분 텍스트를 파싱해서 테이블로 표시 4. 연 2만 건 이상의 데이터를 처리할 수 있도록 100건씩 페이지네이션 5. 테이블 위에 요약 표시: 총 건수, 총 입고량(톤), 총 입고금액(원) 6. "저장" 버튼으로 localStorage에 저장 7. 페이지를 열 때 저장된 데이터가 있으면 자동으로 불러와서 표시 8. 저장 시 "월별 주요 거래처 납품 추이" 차트도 자동 갱신 테이블 디자인: - 헤더는 진한 남색 배경에 흰색 글씨 - 짝수/홀수 행 배경색 구분 - 숫자 열(입고량, 입고금액)은 오른쪽 정렬, 천 단위 콤마

✅ 확인 체크리스트

납품 데이터를 붙여넣고 반영하면 테이블이 보이나요?
페이지 나누기가 동작하나요?
요약 정보(건수, 입고량, 금액)가 표시되나요?
새로고침해도 데이터가 유지되나요?

💬 강사 한마디

2만 건 이상의 대량 데이터도 브라우저에서 처리할 수 있습니다! 핵심은 페이지네이션이에요. 데이터는 전부 저장하되, 화면에는 100건씩만 보여주면 됩니다.

검수 등급 매핑 관리

"품명을 등급으로 자동 변환!"
⏱️ 15분

🧠 이렇게 생각해보세요

1
문제 인식: 납품 데이터에는 "생철A", "중량BS" 같은 세부 품명이 있는데, 보고서에는 "국고상/중/하/선반설" 4개 등급으로 요약해야 합니다. 💭 "생철A → 국고상, 중량BS → 국고중... 이 매핑을 관리하는 기능이 필요하겠다!"
2
필요한 기능 나열하기: "관리"라는 말에는 뭐가 포함될까요? 💭 "보기(테이블) + 추가 + 삭제 + 등급 이동 + 검색 + 저장 + 초기화 → 이것도 하나씩 나열해서 알려주면 됩니다"
3
포인트: 기존 데이터를 구체적으로 알려주세요. "국고상: 생철A, 생철B, 생철AL, 슈레더B"처럼 실제 매핑 데이터를 주면 AI가 바로 기본값으로 넣어줍니다.
📋 AI에게 보낼 프롬프트
"검수 등급 매핑 관리" 기능을 추가해줘. 이 기능의 목적: 거래처 납품 데이터의 "검수등급명(품명)"을 보고 자동으로 국고 등급(국고상/국고중/국고하/선반설)으로 분류하는 매핑 테이블을 관리하는 기능이야. 기본 매핑 데이터: - 국고상: 생철A, 생철B, 생철AL, 슈레더B - 국고중: 중량A, 중량AS, 중AL, 중량 ALC(가위), 중량BS, 모터블럭 - 국고하: 중량B, 경량A, 경량B, 경량L, 길로틴A, 길로틴B, 절단S, 압축A, 압축B, 슈레더C, 중량BLS, 중량C, 경량C, 경량T, 경량TC, 경량S, 중량BLC, 중BL - 선반설: 선반A, 선반C, 압축C, 압축D 구체적으로 이렇게 만들어줘: 1. 등급별로 그룹핑된 매핑 테이블을 보여줘 (국고상/국고중/국고하/선반설 각각 색상 구분) 2. 각 등급 그룹에 새 품명을 추가할 수 있는 입력란과 "추가" 버튼 3. 기존 품명 옆에 "삭제" 버튼으로 매핑 제거 가능 4. 품명의 등급을 다른 등급으로 이동할 수 있는 드롭다운 5. "검색" 입력란: 품명을 입력하면 어떤 등급에 매핑되어 있는지 바로 표시 6. 매핑되지 않은 품명이 납품 데이터에 있으면 "미분류" 경고 표시 7. localStorage에 저장해서 새로고침해도 매핑 데이터 유지 8. "초기화" 버튼으로 기본 매핑 데이터로 되돌리기 가능 9. 거래처 납품 실적 데이터와 연동: 납품 데이터의 검수등급명을 이 매핑 테이블로 자동 분류해서 등급별 입고량/입고금액 요약 표시

💬 강사 한마디

이 프롬프트에서 배울 점: 기능을 1~9번까지 번호로 나열한 것입니다. AI에게 여러 기능을 요청할 때는 이렇게 번호를 매겨서 나열하면 빠뜨리지 않고 만들어줍니다. "잘 만들어줘"보다 훨씬 정확한 결과가 나와요!

😵 화면에 아무것도 안 나와요

에러가 나요. 화면에 아무것도 안 보여요. 코드를 확인해서 고쳐줘.

📊 차트가 안 그려져요

차트가 안 그려져요. Chart.js CDN이 제대로 로드되는지 확인하고, 차트 생성 코드에 에러가 없는지 확인해서 고쳐줘.

📋 데이터 붙여넣기가 안 돼요

엑셀에서 복사한 데이터를 붙여넣고 적용 버튼을 눌러도 반응이 없어요. 탭으로 구분된 텍스트를 파싱하는 부분을 확인하고 고쳐줘.

🎨 디자인이 이상해요 (겹침/너무 작음)

카드들이 겹쳐 보여요 (또는 차트가 너무 작아요, 글씨가 안 보여요). CSS를 확인해서 레이아웃을 고쳐줘.

💾 저장이 안 돼요 (새로고침하면 사라짐)

저장 버튼을 눌러도 새로고침하면 데이터가 사라져요. localStorage에 저장하고 불러오는 부분을 확인해서 고쳐줘.

🤯 코드가 너무 길어서 AI가 중간에 끊겨요
