1

뼈대 만들기

"일단 화면을 띄워보자"
⏱️ 10분

🧠 이렇게 생각해보세요

1
눈으로 그리기: 우리 팀 대시보드를 상상해봅시다 💭 "상단에 로고와 팀명, 그 아래에 메뉴 탭들, 탭을 누르면 내용이 바뀌는 화면"
2
조각으로 나누기: 가장 큰 뼈대부터 잡아봅시다 💭 "① 헤더(로고+팀명) ② 탭 메뉴(부재료실적, 등급별현황) ③ 각 탭의 빈 내용"
3
말로 설명하기: 색상, 위치, 크기를 구체적으로 적어봅시다 💭 "헤더는 진한 남색, 포인트는 오렌지, 배경은 밝은 회색"
4
포인트: 처음부터 완벽하게 안 해도 됩니다. "빈 껍데기"를 먼저 만들고, 나중에 하나씩 채워 넣는 거예요!
📋 AI에게 보낼 프롬프트
동국제강 원료기획팀에서 사용할 업무 대시보드 웹 페이지를 만들어줘. 하나의 HTML 파일로 만들어줘. 상단에 진한 남색 배경의 헤더를 넣고, 왼쪽에 "DK 동국제강" 로고 텍스트, 오른쪽에 오늘 날짜와 "동국제강 원료기획팀 | 이돈석 팀장님" 텍스트를 넣어줘. 헤더 아래에 탭 메뉴 2개를 만들어줘: - 부재료실적 모니터링 - 등급별현황/수입관리 탭을 클릭하면 해당 내용이 보이고, 다른 탭 내용은 숨겨지게 해줘. 지금은 각 탭에 "여기에 OO 내용이 들어갑니다"라고만 표시해줘. 전체 배경은 밝은 회색(#F4F6F9), 헤더는 진한 남색(#1B2A4A), 포인트 색상은 오렌지(#E67E22)로 해줘.

✅ 확인 체크리스트

화면에 네이비색 헤더가 보이나요?
3개 탭을 클릭하면 내용이 바뀌나요?

💬 강사 한마디

지금 결과물은 HTML(구조) + CSS(디자인) + JavaScript(탭 전환 동작)의 조합입니다. AI가 세 가지를 한번에 만들어준 거예요! 이 세 단어를 몰라도 전혀 상관없습니다.

숫자 카드 넣기

"핵심 지표를 한눈에"
⏱️ 10분

🧠 이렇게 생각해보세요

1
눈으로 그리기: 팀장님이 대시보드를 열었을 때 맨 먼저 봐야 할 숫자가 뭘까요? 💭 "올해 목표가 얼마고, 지금 실적이 얼마고, 달성률이 몇 %인지"
2
조각으로 나누기: 핵심 지표를 카드 형태로 정리해봅시다 💭 "카드 4개: 연간목표 | 누계실적 | 달성률 | 거래처 성과율 → 가로로 나란히"
3
포인트: 숫자를 그냥 나열하면 눈에 안 들어옵니다. 큰 글씨 + 작은 설명 + 색상 강조가 핵심이에요. 실무에서 이걸 "KPI 카드"라고 부릅니다.
📋 AI에게 보낼 프롬프트
첫 번째 탭 "부재료실적 모니터링"에 내용을 추가해줘. 맨 위에 숫자 카드 4개를 가로로 나란히 배치해줘: 1. 연간 목표: 272.5만 톤 (설명: 2024년 연간 계획 합계) 2. 누계 실적: 281.5만 톤 (옆에 "초과" 표시를 오렌지색으로) 3. 계획 대비 달성률: 103.3% (옆에 "초과" 표시) 4. 거래처 평균 성과율: 90.4% 카드는 흰색 배경에 둥근 모서리, 큰 숫자는 굵고 크게, 설명은 작은 회색 글씨로 해줘.

✅ 확인 체크리스트

4개 카드가 가로로 나란히 보이나요?
숫자가 크고 눈에 잘 들어오나요?

💬 강사 한마디

프롬프트에 실제 숫자를 넣었더니 AI가 바로 반영했죠? "적당히 넣어줘"보다 "272.5만 톤"처럼 구체적으로 쓸수록 원하는 결과가 나옵니다.

차트 넣기

"숫자를 그림으로"
⏱️ 15분

🧠 이렇게 생각해보세요

1
눈으로 그리기: 숫자 카드만으론 "추이"가 안 보입니다. 월별 흐름을 보려면? 💭 "막대 차트로 계획 vs 실적을 비교하고, 꺾은선 차트로 거래처별 추이를 보면 되겠다"
2
조각으로 나누기: 차트를 배치해봅시다 💭 "왼쪽에 큰 막대 차트(60%), 오른쪽에 작은 꺾은선 차트(40%) → 좁은 화면에서는 위아래로"
3
포인트: 차트의 데이터(숫자)를 직접 알려주세요. AI가 임의로 넣은 데이터는 어차피 나중에 바꿔야 합니다. 우리 실제 계획/실적 숫자를 쓰면 바로 실무용이 됩니다!
📋 AI에게 보낼 프롬프트
KPI 카드 아래에 차트 2개를 나란히 넣어줘. 차트는 Chart.js 라이브러리를 CDN으로 불러와서 사용해줘. 왼쪽 차트 (너비 60%): "월별 계획 대비 실적" - 막대 차트 - 가로축: 1월~12월 - 주황색 막대: 실적 (만톤 단위로 표시) - 파란색 테두리 막대: 계획 - 제목 옆에 "12월 누계 103.3%" 표시 아래 샘플 데이터를 사용해줘: 계획(만톤): 21.5, 21.5, 22.5, 24.5, 24.5, 25.0, 24.0, 20.0, 20.5, 20.0, 25.0, 23.5 실적(만톤): 23.9, 20.6, 22.9, 21.4, 22.6, 25.6, 27.1, 24.7, 22.5, 25.2, 23.1, 21.8 오른쪽 차트 (너비 40%): "월별 주요 거래처 납품 추이" - 꺾은선 차트 - 거래처 3개: 일보앤틱(파란색), 대한강철(주황색), 동국자원(초록색) - 적당한 샘플 데이터를 넣어줘 화면이 좁아지면 차트가 위아래로 배치되게 해줘.

✅ 확인 체크리스트

막대 차트에서 주황(실적)과 파란(계획)이 구분되나요?
꺾은선 차트에 3개 거래처가 각각 다른 색으로 보이나요?

💬 강사 한마디

Chart.js는 인터넷에서 무료로 가져와 쓸 수 있는 차트 도구입니다. CDN이라는 방식으로 인터넷 주소 한 줄이면 사용 가능해요. AI가 알아서 연결해줬습니다.

엑셀 데이터 붙여넣기 ⭐

"진짜 우리 데이터로!" — 오늘의 하이라이트
⏱️ 20분

🧠 이렇게 생각해보세요

1
문제 인식: 지금 차트의 데이터는 코드에 박혀있습니다. 매달 데이터가 바뀌면 어떻게 하죠? 💭 "엑셀에서 데이터를 복사해서 붙여넣으면 차트가 자동으로 바뀌면 좋겠다!"
2
조각으로 나누기: 이 기능을 세부적으로 쪼개봅시다 💭 "① 입력창(textarea) ② 적용 버튼 ③ 저장 버튼 ④ 데이터를 파싱하는 로직 ⑤ 차트 갱신"
3
데이터 형태 설명이 핵심! AI가 데이터를 파싱하려면 "몇 줄이고, 무슨 순서이고, 어떻게 구분되는지"를 정확히 알려줘야 합니다. 💭 "4줄짜리, 탭으로 구분, 순서: 인천계획 → 인천실적 → 포항계획 → 포항실적"
4
+ 거래처 납품 데이터도! 같은 원리로 5열 데이터(전기일/공급업체/검수등급/입고량/입고금액)도 붙여넣기 가능하게 만들어봅시다.
📋 AI에게 보낼 프롬프트
차트 아래에 "월별 계획 대비 실적 데이터" 영역을 추가해줘. 이 영역의 핵심 기능: 엑셀에서 데이터를 복사해서 붙여넣으면, 위의 차트와 KPI 숫자가 자동으로 바뀌는 기능이야. 구체적으로 이렇게 만들어줘: 1. "붙여넣은 값 적용" 버튼과 "저장" 버튼을 만들어줘 2. 버튼을 누르면 큰 텍스트 입력창(textarea)이 나타나게 해줘 3. 사용자가 엑셀에서 아래 형태의 데이터를 복사해서 붙여넣어: 4줄짜리 데이터 (탭으로 구분됨) 1줄: 인천 계획 숫자 12개 (1월~12월) 2줄: 인천 실적 숫자 12개 3줄: 포항 계획 숫자 12개 4줄: 포항 실적 숫자 12개 4. "적용" 버튼을 누르면 붙여넣은 텍스트를 줄바꿈과 탭으로 나눠서 숫자로 변환하고, 인천+포항 합계로 차트와 KPI 카드를 자동 갱신해줘 5. "저장" 버튼을 누르면 localStorage에 저장해서 새로고침해도 데이터가 유지되게 해줘 6. 페이지를 열 때 localStorage에 저장된 데이터가 있으면 자동으로 불러와줘 아래에 현재 데이터를 테이블로도 보여줘 (인천 계획, 인천 실적, 포항 계획, 포항 실적 4줄) 그리고 그 아래에 "거래처 납품 실적관리 데이터" 영역도 만들어줘: - 큰 텍스트 입력창에 엑셀 5개 열을 복사해 붙여넣기 (전기일 / 공급업체명 / 검수등급명 / 입고량 / 입고금액) - "원본 데이터 반영" 버튼으로 데이터를 파싱해서 테이블로 표시 - "저장" 버튼으로 localStorage에 저장 - 데이터가 많으니까 100개씩 페이지 나누기

✅ 확인 체크리스트

텍스트 입력창에 위 숫자를 붙여넣고 적용하면 차트가 바뀌나요?
새로고침해도 데이터가 유지되나요?

💬 강사 한마디

localStorage는 브라우저 안의 개인 서랍장 같은 거예요. 서버 없이도 데이터를 저장할 수 있습니다. 그리고 프롬프트에서 "4줄짜리, 탭으로 구분"이라고 데이터 형태를 명확히 알려준 것이 핵심입니다. AI는 이 설명을 보고 데이터를 파싱하는 코드를 만든 거예요.

검수 등급 매핑 관리

"품명을 등급으로 자동 변환!"
⏱️ 15분

🧠 이렇게 생각해보세요

1
문제 인식: 납품 데이터에는 "생철A", "중량BS" 같은 세부 품명이 있는데, 보고서에는 "국고상/중/하/선반설" 4개 등급으로 요약해야 합니다. 💭 "생철A → 국고상, 중량BS → 국고중... 이 매핑을 관리하는 기능이 필요하겠다!"
2
필요한 기능 나열하기: "관리"라는 말에는 뭐가 포함될까요? 💭 "보기(테이블) + 추가 + 삭제 + 등급 이동 + 검색 + 저장 + 초기화 → 이것도 하나씩 나열해서 알려주면 됩니다"
3
포인트: 기존 데이터를 구체적으로 알려주세요. "국고상: 생철A, 생철B, 생철AL, 슈레더B"처럼 실제 매핑 데이터를 주면 AI가 바로 기본값으로 넣어줍니다.
📋 AI에게 보낼 프롬프트
